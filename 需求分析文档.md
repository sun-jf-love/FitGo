# FitGo - 每日运动提醒App 需求分析文档

## 1. 项目概述

**项目名称**: FitGo  
**项目类型**: 移动端应用 (uni-app X)  
**项目定位**: 轻量级每日运动提醒与记录工具，帮助用户养成规律运动习惯  

**核心功能**: 每日运动提醒 + 多种运动类型记录 + 本地数据存储与导出

---

## 2. 用户需求分析

### 2.1 核心需求

| 需求类型 | 优先级 | 描述 |
|---------|-------|------|
| 每日提醒 | P0 | 通过系统推送通知 + App内弹窗双重提醒用户运动 |
| 运动记录 | P0 | 支持记录多种运动类型（跑步、瑜伽、力量训练等） |
| 数据存储 | P0 | 本地存储运动数据，支持导出备份 |
| 打卡统计 | P1 | 展示每日/每周/每月运动打卡情况 |
| 目标设置 | P1 | 用户可设置每日运动目标（如运动时长、次数） |

### 2.2 用户画像

- **目标用户**: 通用（适用于健身初学者至有一定经验的用户）
- **使用场景**: 
  - 每天固定时间提醒运动
  - 运动完成后快速记录
  - 查看历史运动数据
  - 数据备份导出

---

## 3. 功能模块设计

### 3.1 提醒模块

- **系统推送通知**
  - 用户可设置提醒时间（如每天早上8:00）
  - 支持工作日/周末自定义
  - 推送内容可自定义文案
  
- **App内弹窗提醒**
  - 定时弹出提醒窗口
  - 支持稍后提醒、立即开始、跳过今日

### 3.2 运动记录模块

- **运动类型**（预设常见类型，可扩展）
  - 跑步
  - 瑜伽
  - 力量训练
  - 游泳
  - 骑行
  - 健身操
  - 其他（自定义）

- **记录内容**
  - 运动类型选择
  - 运动时长（分钟）
  - 运动日期
  - 可选：消耗卡路里、备注

### 3.3 数据管理模块

- **本地存储**
  - 使用 uni-app 本地存储 API
  - 结构化存储运动记录
  
- **数据导出**
  - 支持导出为 JSON/CSV 格式
  - 导出到手机本地

### 3.4 统计与展示模块

- **首页展示**
  - 今日运动状态（已完成/未完成）
  - 本周运动天数统计
  - 连续打卡天数
  
- **历史记录**
  - 按日期查看运动记录
  - 按运动类型筛选

### 3.5 设置模块

- 提醒时间设置
- 提醒方式选择（推送/弹窗/两者）
- 运动目标设置（每日运动时长目标）
- 数据导出功能
- 关于

## 4. 页面结构

```
pages/
├── index/index          # 首页（今日状态、快捷记录、快速打卡）
├── record/record        # 运动记录页面（记录运动、查看记录）
├── stats/stats          # 统计页面（打卡统计、数据图表）
└── settings/settings    # 设置页面（提醒、目标、导出）
```

## 5. 数据结构设计

### 5.1 运动记录结构

```json
{
  "id": "uuid",
  "type": "跑步",
  "duration": 30,
  "calories": 300,
  "date": "2024-01-15",
  "remark": "晨跑",
  "createdAt": "timestamp"
}
```

### 5.2 用户设置结构

```json
{
  "reminderTime": "08:00",
  "reminderDays": [1,2,3,4,5],  // 周一至周五
  "reminderMethod": "both",      // both/push/inapp
  "dailyGoal": 30,              // 每日运动时长目标（分钟）
  "sportsTypes": ["跑步", "瑜伽", "力量训练"]
}
```

## 6. 技术实现要点

- **uni-app X**: 使用 UTS 开发，支持多端编译（App-Android, 鸿蒙, 微信小程序等）
- **本地存储**: 使用 uni.setStorage / uni.getStorage
- **消息推送**: 使用 uni-app 推送相关 API
- **时间选择**: 使用 picker 组件
